Bu adımlar, siteni bir "AI Ajanı" (ChatGPT, Gemini veya otonom botlar) için anlaşılabilir kılmak adına harika bir yol haritası. Bir yazılımcı değilsen bu biraz karmaşık gelebilir, ancak adım adım ne yapman gerektiğini aşağıda özetledim.

Siteni bir **AI mıknatısına** dönüştürmek için şu sırayı izleyebilirsin:

---

### 1. Adım: API Kataloğu Oluşturmak (Haritayı Çizmek)

AI ajanları sitende hangi "kapıların" (API servislerinin) olduğunu bilmek ister. Bunun için sitenin kök dizinine bir dosya eklemelisin.

* **Ne yapmalı:** Sitenin sunucusunda `/.well-known/api-catalog` adında bir dosya oluştur.
* **İçeriği (JSON formatında):**

```json
{
  "linkset": [
    {
      "anchor": "https://nettotoboya.com/api",
      "service-desc": [ { "href": "/api/openapi.json", "type": "application/json" } ],
      "service-doc": [ { "href": "/docs/api", "type": "text/html" } ],
      "status": [ { "href": "/api/health", "type": "application/json" } ]
    }
  ]
}

```

* **Sunucu Ayarı:** Bu dosyanın `Content-Type: application/linkset+json` başlığıyla sunulduğundan emin olmalısın.

---

### 2. Adım: HTTP Link Header Ekleme (Tabela Asmak)

Botlar sitene geldiğinde ilk olarak HTTP başlıklarına (header) bakar. Onlara "Hey, benim bir kataloğum var!" demen lazım.

* **Ne yapmalı:** Sunucu konfigürasyonuna (Nginx veya Apache gibi) şu satırı ekle:
* **Nginx kullanıyorsan:**
`add_header Link '</.well-known/api-catalog>; rel="api-catalog", </docs/api>; rel="service-doc"';`
* **Cloudflare kullanıyorsan:** "Transform Rules" kısmından HTTP Response Header olarak bunu ekleyebilirsin.

---

### 3. Adım: Markdown Desteği (AI'nın Dilini Konuşmak)

AI'lar karmaşık HTML kodlarını okumaktansa sade Markdown metinlerini çok daha iyi anlar. Bot "Bana bu sayfayı Markdown olarak ver" dediğinde (Accept: text/markdown), siten buna cevap vermeli.

* **Nasıl Çalışır:** Backend kodunda (PHP, Node.js, Python vb.) gelen isteği kontrol etmelisin.
* **Mantık:**
1. İstek başlığında `Accept: text/markdown` var mı?
2. Varsa: Sayfanın içeriğini Markdown olarak döndür.
3. Yanıt başlığına şunu ekle: `Content-Type: text/markdown`.



---

### 4. Adım: OAuth Discovery (Güvenli Giriş Kapısı)

Eğer sitende randevu almak veya fiyat teklifi görmek için bir giriş (login) sistemi varsa, botun buraya nasıl gireceğini bilmesi gerekir.

* **Ne yapmalı:** `/.well-known/oauth-protected-resource` dosyası oluştur ve içine şunları yaz:

```json
{
  "resource": "https://nettotoboya.com/api",
  "authorization_servers": ["https://auth.nettotoboya.com"],
  "scopes_supported": ["read", "write", "appointments"]
}

```

---

### 5. Adım: WebMCP Uygulaması (Tarayıcı Araçları)

Bu en yenisidir. Eğer bir kullanıcı senin siteni AI destekli bir tarayıcıda (Chrome'un yeni versiyonları gibi) açarsa, AI sitendeki araçları (fiyat hesaplama, randevu formu gibi) doğrudan kullanabilir.

* **Ne yapmalı:** Sitenin ana JavaScript dosyasına şu kodu ekle:

```javascript
if (navigator.modelContext && navigator.modelContext.provideContext) {
  navigator.modelContext.provideContext({
    tools: [{
      name: "get_paint_quote",
      description: "Oto boya fiyat teklifi al",
      inputSchema: {
        type: "object",
        properties: {
          car_model: { type: "string", description: "Arabanın markası ve modeli" },
          part: { type: "string", description: "Boyanacak parça (kapı, tampon vb.)" }
        }
      },
      execute: async (args) => {
        // Burada senin fiyat hesaplama fonksiyonun çalışacak
        return { price: "2500 TL", duration: "2 gün" };
      }
    }]
  });
}

```

---

### Özet: Bunlar Sana Ne Kazandırır?

1. **Arama Motoru Avantajı:** Google ve diğer AI botları siteni "yüksek kaliteli ve yapılandırılmış" olarak işaretler.
2. **Siri/ChatGPT/Gemini Uyumu:** Birisi "Antalya'da boyacı bul ve randevu al" dediğinde, AI senin sitendeki formu senin adına doldurabilir hale gelir.
3. **Hız:** AI ajanları siteni saniyeler içinde tarayıp analiz eder.

**Tavsiye:** Eğer bir yazılımcıyla çalışıyorsan, yukarıdaki listeyi ona doğrudan iletebilirsin. Bunlar güncel web standartları olduğu için çoğu modern geliştirici bu terimlere (RFC 8288, JSON-LD, OAuth) aşinadır.
